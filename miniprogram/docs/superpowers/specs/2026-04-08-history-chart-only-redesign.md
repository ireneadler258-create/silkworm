# 历史页面曲线图重构设计（仅图表链路）

## 目标
仅重构历史页面中的曲线图链路，解决以下问题：

1. `1h` / `6h` 范围下曲线图经常显示不出来或看起来像没有加载
2. `24h` / `7d` 进入后需要明显向左滑动才能看到主要内容
3. 图表整体体验不够像成熟商业级 App
4. 不修改告警列表、弹窗、其他页面和非图表业务模块

## 范围
### 可修改
- `src/components/LineChart.vue`
- `src/pages/history/index.vue` 中与图表相关的容器、传参、图表接线逻辑
- 图表使用时的少量数据整理逻辑（仅服务图表）

### 不修改
- 历史页中的告警记录列表
- 历史页详情弹窗业务逻辑
- 其他页面
- store 中与非图表业务相关的逻辑

## 当前问题判断

### 1. 1h / 6h 显示不出来
这更像是图表布局模型和低样本渲染策略的问题，而不只是“没有数据”。

当前实现中：
- 图表对少量点的可视化策略不稳定
- 状态提示层与 canvas 绘制叠加后，容易产生“像没画出来”的观感
- 现有滚动与宽度策略对短时间范围并不友好

### 2. 24h / 7d 需要猛向左滑
这是当前图表采用“横向滚动画布 + 不理想初始视口”的典型结果：
- 画布内容起始布局与用户预期不一致
- 首屏展示区域没有落在用户最需要看到的有效趋势区段
- 用户需要先做一次明显拖动，才能进入可读状态

### 3. 商业级体验不足
当前图表的主要短板不只是风格，而是图表系统层面的体验不完整：
- 首屏不可读
- 点选反馈不够稳定
- 折线、网格、面积、阈值线的层级不够成熟
- 图表在不同时间范围下没有采用最合适的呈现模式

## 重构策略
采用 **“图表双模式 + 接线最小改动”** 的方案。

### 模式 A：固定视口图表
用于：
- `1h`
- `6h`

策略：
- 不允许横向滚动
- 一屏直接展示完整曲线
- 即使只有少量数据点，也要保证能稳定看见主线和关键点
- 适合快速查看最近变化

### 模式 B：受控横向图表
用于：
- `24h`
- `7d`

策略：
- 允许横向扩展
- 首屏自动落到正确可读区域
- 用户仍可滑动查看更多数据，但不需要先“猛滑一下”才能看见内容
- 自动控制点位密度，保证趋势可读性

## 图表架构设计

### 一、数据输入层
由历史页传给图表的数据保持简洁，只包含图表需要的：
- categories
- values
- 当前 range 信息
- 阈值信息
- 可选状态文案

历史页只负责：
- 提供当前指标数据
- 传入时间范围
- 传入图表模式

图表组件内部负责：
- 计算画布宽度
- 计算首屏视口策略
- 计算点位坐标
- 处理 tooltip / 选中态 / 阈值绘制

### 二、布局层
重构重点是重新定义画布布局：

#### 短范围（1h / 6h）
- 画布宽度固定为容器宽度
- 不依赖横向滚动
- 使用“压缩但不失真”的点距策略

#### 长范围（24h / 7d）
- 画布宽度随点数增加，但有合理上限/节奏
- 首次进入时自动定位到最合理的可视区域
- 保证不出现“必须猛向左滑一次才正常”的体验

### 三、渲染层
图表渲染层重写为更成熟的层级：
- 背景层
- 网格层
- 阈值线层
- 面积层
- 主折线层
- 关键点/异常点层
- 选中指示层
- Tooltip 层

重点要求：
- 主线更清晰
- 面积渐变更克制
- 网格更轻但可读
- 阈值线不抢主线
- 异常点可见但不杂乱

### 四、交互层
交互按成熟商业图表方式处理：
- 点击或触摸时吸附最近点
- 高亮当前点
- 显示稳定 tooltip
- tooltip 中展示准确时间和值
- 点选映射必须准确，不允许“点 A 显示成 B”

## 视觉方向
目标不是炫技，而是专业、稳定、清晰。

会采用：
- 更清楚的主线对比度
- 更自然的面积过渡
- 更轻的辅助网格
- 更高级的点选反馈
- 更产品化的 tooltip 视觉

要求与现有历史页视觉兼容，但图表本身提升到接近成熟 IoT / 数据监控 App 的水平。

## 关键设计要求
1. `1h` 打开即见图
2. `6h` 打开即见图
3. `24h` 打开首屏即合理
4. `7d` 打开首屏即合理
5. 点按查看必须准确
6. 不依赖用户先做大幅滑动操作
7. 不改非图表业务区域

## 验证标准
### 功能验证
- 切到 `1h`：图表可见，主线明确
- 切到 `6h`：图表可见，主线明确
- 切到 `24h`：首屏无需明显左滑即可看到主要内容
- 切到 `7d`：首屏无需明显左滑即可看到主要内容
- 点击点位：tooltip 内容与点位对应正确

### 体验验证
- 图表比当前版本更稳定
- 图表比当前版本更像商业级 App
- 图表不会出现“看起来没加载”的错觉

## 不做的事
为保证范围清晰，本次不做：
- 引入新图表库
- 重构整个历史页
- 修改告警记录区
- 修改弹窗业务
- 修改其他页面

## 预期结果
完成后，历史页的曲线图会成为一个更独立、更成熟的图表模块：
- 短时间范围稳定直接可见
- 长时间范围首屏合理可读
- 交互准确
- 视觉更专业
- 同时仍保持当前项目的页面结构不被波及
